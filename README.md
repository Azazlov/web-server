# FileShare Local v2.1

Локальный файловый сервер с авторизацией и разграничением прав доступа.

## Возможности

- **Система ролей**: Администратор и Пользователь
- **Общая папка**: Полный доступ для админа, только чтение для пользователей
- **Личные папки**: Полная приватность для каждого пользователя
- **Иерархия папок**: Поддержка вложенных папок
- **Веб-интерфейс**: Адаптивный дизайн на Bootstrap 5
- **Безопасность**: Хеширование паролей, проверка прав доступа
- **Админ-панель**: Управление пользователями, настройками и сервером
- **Модульная архитектура**: Разделение на независимые компоненты

## Быстрый старт

### Windows

1. **Запуск от имени администратора:**
   ```batch
   cd scripts
   setup_and_run.bat
   ```

2. **Откройте браузер:**
   - http://fileserver.local
   - http://localhost

### Linux/macOS

1. **Запуск от root:**
   ```bash
   cd scripts
   sudo ./setup_and_run.sh
   ```

2. **Откройте браузер:**
   - http://fileserver.local
   - http://localhost

## Архитектура модулей

### Application Factory (`app.py`)

```python
def create_app():
    app = Flask(__name__)
    
    # Конфигурация
    app.config['SECRET_KEY'] = '...'
    app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///database.db'
    
    # Расширения
    db.init_app(app)
    login_manager.init_app(app)
    
    # Регистрация blueprints
    app.register_blueprint(auth_bp)
    app.register_blueprint(main_bp)
    app.register_blueprint(files_bp)
    app.register_blueprint(admin_bp)
    
    return app
```

```bash

```

```python
app.run(host='0.0.0.0', port=8080, debug=False)
```

## Первое использование

1. Запустите сервер
2. Перейдите на http://fileserver.local
3. Нажмите **Зарегистрироваться**
4. Первый пользователь автоматически становится **Администратором**
5. Последующие пользователи будут обычными **Пользователями**

## Админ-панель

Администратору доступна панель управления (**Админ-панель** в меню):

- **Настройки сервера** — информация о порте, размере файла, статусе регистрации
- **Перезапуск сервера** — кнопка перезапуска без остановки вручную
- **Регистрация пользователей** — включение/выключение возможности регистрации
- **Управление пользователями** — таблица всех пользователей с возможностью:
  - Заблокировать/разблокировать загрузку файлов в личную папку
  - Просмотр ролей и дат регистрации
- **Статистика** — количество пользователей, администраторов, обычных пользователей

## Настройка брандмауэра

### Windows (PowerShell от администратора)

```powershell
New-NetFirewallRule -DisplayName "FileShare Local" -Direction Inbound -LocalPort 80 -Protocol TCP -Action Allow
```

### Linux (UFW)

```bash
sudo ufw allow 80/tcp
```

## Доступ из локальной сети

1. Узнайте IP-адрес сервера:
   - Windows: `ipconfig`
   - Linux/Mac: `ifconfig` или `ip addr`

2. Другие пользователи могут получить доступ по адресу:
   ```
   http://YOUR_IP_ADDRESS
   ```

3. Для доступа по домену добавьте в hosts файл клиентов:
   - Windows: `C:\Windows\System32\drivers\etc\hosts`
   - Linux/Mac: `/etc/hosts`
   
   ```
   YOUR_IP_ADDRESS fileserver.local
   ```

## Файл конфигурации

Настройки сервера хранятся в файле `config.json` и сохраняются при изменении через админ-панель:

```json
{
    "server_port": 80,
    "max_upload_size_mb": 100,
    "allow_registration": true,
    "debug_mode": false
}
```

**Параметры:**

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `server_port` | Порт для запуска сервера | 80 |
| `max_upload_size_mb` | Максимальный размер файла (МБ) | 100 |
| `allow_registration` | Разрешить регистрацию пользователей | true |
| `debug_mode` | Режим отладки Flask | false |

**Изменение настроек:**

1. **Через админ-панель** (рекомендуется):
   - Настройки сохраняются автоматически в `config.json`
   - Требуется перезапуск сервера для применения некоторых изменений

2. **Прямое редактирование `config.json`**:
   - Откройте файл в текстовом редакторе
   - Измените нужные параметры
   - Перезапустите сервер

## Устранение проблем

### Ошибка "Address already in use"

Другой сервис использует порт 80. Варианты решения:

1. **Остановить другой сервис:**
   - Windows: `netstat -ano | findstr :80` для поиска процесса
   - Linux/Mac: `lsof -i :80` для поиска процесса

2. **Изменить порт в `config.json`:**
   ```json
   {
       "server_port": 8080
   }
   ```

3. **Использовать скрипт перезапуска:**
   - Windows: `scripts/restart_server.bat`
   - Linux/Mac: `scripts/restart_server.sh`

### Перезапуск сервера через админ-панель

**Известная проблема:** После автоматического перезапуска может появиться ошибка "Address already in use", так как старый процесс ещё держит порт.

**Решение:**

1. **Автоматический перезапуск (Linux/Mac):**
   ```bash
   scripts/restart_server.sh
   ```

2. **Автоматический перезапуск (Windows):**
   ```batch
   scripts\restart_server.bat
   ```

3. **Ручной перезапуск:**
   - Остановите текущий процесс (Ctrl+C)
   - Запустите снова: `python app.py` или через `scripts/setup_and_run.bat`

### Ошибка доступа при запуске

- Windows: Запустите от имени администратора
- Linux/Mac: Используйте `sudo`

### Не доступен с других устройств

1. Проверьте брандмауэр
2. Убедитесь, что сервер слушает `0.0.0.0`
3. Проверьте IP-адрес сервера

### Сброс базы данных

Удалите `database.db` для сброса всех пользователей:

```bash
rm database.db
```

### Сброс настроек

Удалите `config.json` для сброса настроек к значениям по умолчанию.
